Mengatasi Error "View... Not Found" saat Setup React Native di Android Studio

Irfan Agung Syaputra

Mahasiswa Informatika • 2024

Halo semuanya! Kenalin, saya Irfan Agung Syaputra, seorang mahasiswa Informatika yang saat ini lagi seru-serunya menyelami dunia mobile development.

Sebagai mahasiswa IT, kita semua tahu kalau bagian paling menantang dari belajar framework baru itu bukan pas ngodingnya, melainkan saat proses instalasi dan setup environment. Setuju? 😂

Baru-baru ini, pas saya lagi mencoba setup proyek React Native menggunakan Android Studio, saya menabrak sebuah error yang cukup bikin garuk-garuk kepala. Pesannya kira-kira seperti ini di terminal:

Task :app:compileDebugJavaWithJavac FAILED
An exception has occurred in the compiler... View... Not Found / SDK location not found.

Kalau kamu lagi mengalami hal yang sama, tenang, kamu gak sendiri. Di artikel ini, saya mau bagikan cara praktis mengatasinya berdasarkan pengalaman saya.

🛠️ Mengapa Error Ini Terjadi?

Biasanya, React Native gagal melakukan build ke emulator atau perangkat Android karena dia tidak tahu di mana letak Android SDK di dalam komputer kita. Masalah ini sering muncul karena kita lupa mengatur Environment Variables di sistem operasi atau ada konfigurasi Gradle yang terlewat.

🚀 Langkah demi Langkah Menyelesaikannya

Langkah 1: Buat File local.properties (Cara Paling Ampuh)

Seringkali, React Native butuh petunjuk langsung di dalam folder proyek kita.

  1. Buka folder proyek React Native kamu menggunakan VS Code.
  2. Masuk ke dalam folder android (nama-proyek/android/).
  3. Buat file baru di dalam folder tersebut dengan nama local.properties.
  4. Salin baris kode di bawah ini (sesuaikan dengan nama user PC/Laptop kamu):
# Untuk pengguna Windows
sdk.dir=C:\\Users\\NAMA_USER_KAMU\\AppData\\Local\\Android\\Sdk

# Untuk pengguna macOS
sdk.dir=/Users/NAMA_USER_KAMU/Library/Android/sdk

Langkah 2: Cek Environment Variables (Windows)

Jika langkah pertama belum cukup, pastikan sistem operasi kamu sudah mengenali Android HOME.

  • Cari "Edit the system environment variables" di Windows Search.
  • Klik tombol Environment Variables.
  • Di bagian User variables, tambahkan variable baru dengan nama ANDROID_HOME dan value path SDK Anda.

Langkah 3: Bersihkan Cache Gradle (Clean Build)

Setelah konfigurasi di atas selesai, kita perlu membersihkan sisa-sisa build yang error sebelumnya.

cd android
./gradlew clean
cd ..
npx react-native run-android

Kesimpulan: Error saat pertama kali setup environment itu adalah makanan sehari-hari anak Informatika. Kuncinya adalah jangan panik dan baca log error-nya pelan-pelan!
